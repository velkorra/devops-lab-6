# Load Tester - нагрузочный тестер на Go

Утилита для нагрузочного тестирования HTTP-сервисов. Построена на библиотеке [Vegeta](https://github.com/tsenart/vegeta) (v12) и выводит детальную статистику на русском языке: RPS, латентность, перцентили, графики по интервалам и отчёт в файл.

## Требования

- Go 1.21+

## Установка зависимостей

```bash
cd load-tester
go mod download
```

## Запуск

```bash
go run .
```

С флагами (все опциональны):

| Флаг | По умолчанию | Описание |
|------|--------------|----------|
| `-url` | `http://localhost:8080/work` | URL для тестирования |
| `-rps` | `100` | Целевое количество запросов в секунду |
| `-duration` | `30s` | Длительность теста (например `10s`, `1m`, `2m`) |
| `-interval` | `1s` | Интервал для детальной статистики по временным отрезкам |
| `-timeout` | `30s` | Таймаут одного HTTP-запроса |
| `-method` | `GET` | HTTP-метод |
| `-body` | - | Путь к файлу с телом запроса (для POST/PUT и т.д.) |
| `-headers` | - | Заголовки в формате `Key: Value, Key2: Value2` |

### Примеры

```bash
# Базовый тест: 50 req/s в течение 1 минуты
go run . -url http://localhost:8080/work -rps 50 -duration 1m

# Короткий тест с высоким RPS
go run . -rps 200 -duration 10s

# POST с телом и заголовком
go run . -url http://localhost:8080/api -method POST -body payload.json -headers "Content-Type: application/json"
```

## Что выводит программа

1. **Во время теста** - прогресс: процент выполнения, число запросов, количество активных запросов, текущая латентность и фактический RPS за последнюю секунду.

2. **Основные результаты**  
   Время начала/окончания, длительность, всего запросов, успешных (2xx), процент успешности, целевой и реальный RPS, средняя/медиана/P95/P99/мин/макс латентности.

3. **Статистика по интервалам**  
   Таблица по каждому интервалу (по умолчанию 1 с): запросов, успешных, % успеха, средний/P95/P99 latency, RPS и % от целевого RPS (с цветовой индикацией).

4. **Графики в консоли**  
   - RPS по интервалам (полоски и фактический RPS).  
   - P95 латентность по интервалам.

5. **Перцентили**  
   P50, P75, P90, P95, P99, P99.9, P100 с визуализацией.

6. **Анализ производительности**  
   Сравнение целевого и фактического числа запросов, реальный RPS, разница в процентах.

7. **Анализ ошибок**  
   Распределение HTTP-статусов (2xx, 3xx, 4xx, 5xx) с подсветкой в терминале.

8. **Файл отчёта**  
   Текстовый отчёт сохраняется в текущую директорию: `load_test_report_YYYY-MM-DD_HH-MM-SS.txt` (дата/время окончания теста).

## Сборка бинарника

```bash
go build -o load-tester .
```

Запуск:

```bash
./load-tester -url http://localhost:8080/work -rps 20 -duration 30s
```

## Использование с Quarkus-демо

В корне репозитория описан Quarkus-сервис с лимитом запросов в минуту (`app.max-requests-per-minute`). После запуска сервиса (`./gradlew quarkusDev`) можно:

1. Узнать лимит: `curl http://localhost:8080/work/info`
2. Запустить нагрузчик с RPS ниже лимита - большинство ответов 200.
3. Увеличить `-rps` выше лимита - появятся 429 (Too Many Requests) и это будет видно в блоке «Анализ ошибок» и в отчёте.

Так можно наглядно показать необходимость масштабирования при росте нагрузки.
